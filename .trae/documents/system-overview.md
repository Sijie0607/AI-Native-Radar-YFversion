# AI-Native 读书雷达 - 系统概述

## 1. 产品定位

### 1.1 一句话定位
AI-Native 读书雷达：AI 从业者共建的动态读书雷达，通过专业可信的集体推荐与评分，帮助用户快速发现高质量AI学习资料。

### 1.2 详细定位说明
- **起源**：最初为3000人科技组织做AI人才能力提升规划时，客户期望提供学习资料库，同时内部也缺少体系化的AI学习资料参考。
- **核心理念**：汇聚每个人的专业知识，形成完整的知识地图
- **价值主张**：
  - 专业可信：由AI从业者专业推荐和评分
  - 体系化：按8大领域和难度层级组织
  - 动态演化：雷达图可视化，高价值内容
  - 长期价值：推荐不易过期、具有业界影响力的书籍
  - 与时俱进：体现AI发展趋势，追本溯源

### 1.3 愿景目标
- 短期目标：成为AI学习资料参考平台，提供有效的AI学习路径
- 长期目标：成为AI业界高价值、高信息、高水平的交流平台，企业文化、企业形象和企业品牌的一部分，打出业界影响力

### 1.4 核心价值理念
- 更重视书本身的长期价值
- 与时俱进，体现AI发展趋势
- 追本溯源，推荐不易过期、具有业界影响力的书籍

## 2. 目标用户（Persona）

### 2.1 AI 产品经理
- **身份**：张明，工作1-5年的AI产品经理
- **核心目标**：快速找到高质量、适合当前水平的AI相关书籍，避免踩坑
- **典型场景**：
  - 想入门Agent设计，但不知道从哪本书开始
  - 工作中遇到数据智能相关问题，需要找权威参考书
  - 想了解AI商业落地的最新实践
- **主要痛点**：
  - 书籍太多，不知道哪些真的有价值
  - 网上推荐零散，缺乏系统性分类
  - 难以判断书籍难度是否适合自己
- **当前替代方案**：刷知乎/豆瓣书单、问同事朋友推荐、看出版社/作者名气选书

### 2.2 入门从业者
- **身份**：李华，刚进入AI领域的新人
- **核心目标**：建立AI学习路径，快速入门AI领域
- **典型场景**：
  - 从0基础想学习AI，但不知道从哪开始
  - 需要体系化学习某个AI知识，而不是零散学习
  - 希望获得专业推荐，避免走弯路
- **主要痛点**：信息过载，不知道从何入手
- **当前替代方案**：网上各种课程，但缺乏系统性

### 2.3 跨领域从业者
- **身份**：王芳，工作非强相关但有志于AI领域的人
- **核心目标**：在自己行业应用AI技术
- **典型场景**：希望了解AI在自己行业的应用案例和实践
- **主要痛点**：找不到适合自己行业的AI学习资料
- **当前替代方案**：网上搜索，但信息分散

### 2.4 技术从业者
- **身份**：赵强，想进入该领域的技术人员
- **核心目标**：深入学习某个AI技术方向
- **典型场景**：
  - 学习LLM应用技术选型
  - Agent系统设计
  - 语义检索和RAG技术
- **主要痛点**：技术更新太快，需要高质量技术书籍良莠不齐
- **当前替代方案**：看技术文档和论文，但不够系统化

## 3. 核心使用场景

### 3.1 场景一：探索发现学习资料
- **用户**：所有想学习AI知识的人
- **场景描述**：用户访问AI-Native读书雷达，通过雷达图可视化探索不同领域的学习资料
- **用户需求**：快速了解有哪些高质量的AI学习资料
- **使用流程**：
  1. 用户访问首页，看到雷达图
  2. 在不同领域的资料分布
  3. 悬停查看资料摘要
  4. 点击进入详情页
  5. 查看完整信息和推荐理由
- **痛点**：之前信息分散，难以找到高质量资料
- **价值**：通过雷达图直观发现高质量资料，快速了解AI学习路径

### 3.2 场景二：筛选查找特定领域深入学习
- **用户**：有特定学习目标的用户
- **场景描述**：用户想学习某个特定领域，筛选该领域的资料
- **用户需求**：找到特定领域的高质量学习资料
- **使用流程**：
  1. 用户进入列表页
  2. 选择想要学习的领域
  3. 设置推荐指数
  4. 浏览列表中的资料
  5. 查看详情并选择阅读
- **痛点**：之前网上推荐零散，缺乏系统性
- **价值**：按领域和推荐指数筛选，快速找到高质量资料

### 3.3 场景三：根据能力提升学习路径
- **用户**：想体系化提升AI能力的用户
- **场景描述**：用户想提升自己的AI能力，通过能力主题找到学习路径
- **用户需求**：体系化提升自己的AI能力
- **使用流程**：
  1. 用户查看能力主题
  2. 找到相关资料
  3. 按难度从内到外学习
- **痛点**：之前缺乏体系化的学习路径
- **价值**：提供体系化的AI能力提升路径

## 4. 领域划分

### 4.1 8大领域（雷达图分区）
1. **AI技术工程**
2. **AI产品设计**
3. **Agent与智能体**
4. **AI组织变革**
5. **数据智能与知识**
6. **AI商业落地**
7. **AI伦理治理**
8. **AI前沿趋势**

### 4.2 能力主题（精细化标签）
- **全员通用**：通用AI素养与AI fluency 4D框架、Prompt工程基础与进阶、知识工程与Context管理
- **技术-应用**：AI辅助开发、LLM应用技术选型与架构设计、Agent系统设计与多智能体架构、AI系统与Agent评估监测
- **技术-数据**：语义检索系统设计与RAG、实时数据流与AI集成、面向AI的数据治理、业务知识建模与知识图谱构建
- **技术-平台**：LLMOps平台设计与模型全生命周期管理、AgentOS平台搭建与Agent运行时、模型推理优化与加速、AI基础设施运维与可观测性、AI安全治理与AI Governance
- **产品**：AI产品需求分析与场景探索、AI产品设计与人机协作交互设计、快速原型验证与Vibe Coding
- **测试**：AI辅助测试工程与LLM/Agent测试评估、AI系统专项质量验证、智能质量门禁与测试左移
- **项目管理**：AI辅助项目管理与项目管理数字化工具、AI项目组合管理与数据驱动决策
- **FDE**：业务流程建模与重构、AI应用生产部署与交付实施、流程变革推动与变革管理

### 4.3 难度层级
- 内圈：入门级（难度1-2）
- 中圈：进阶级（难度3）
- 外圈：高级（难度4-5）

- 范围：3-5星
- 说明：只推荐高质量资料，最低3星起评
- 来源：由专业AI从业者推荐和评分
